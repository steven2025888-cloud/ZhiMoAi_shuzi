# 织梦AI 数字人平台 — 项目目录结构及代码说明

## 项目概述

织梦AI 是一个数字人视频合成平台，包含 **Windows 桌面客户端**、**手机端 App**、**后端 API 服务** 和 **GPU 推理服务** 四大模块。

**整体架构:**

```
手机端(hb_ip) ──┐
                ├──→ 后端API(Hyperf/PHP) ──WebSocket──→ GPU后端(HeyGem/Python)
桌面端(Gradio) ─┘         ↕                                    ↕
                      MySQL/Redis                        GPU 推理引擎
```

### 手机端通信架构

- **HTTP 请求** → PHP API 端 (`api.zhimengai.xyz`)
  - 卡密认证、TTS 合成、资产管理
- **WebSocket** → Hyperf WS (`wss://api.zhimengai.xyz/dsp`)
  - GPU 状态推送（上线/离线）、任务结果实时回传
- **直连 GPU** → GPU 合成服务器 (`run_server.py`)
  - HeyGem 视频合成、文件上传（GPU 在线时优先直连，离线走 PHP 代理排队）

### GPU 后端核心流程

```
音频文件 + 数字人视频
        ↓
  trans_dh_service.TransDhTask.work()
        ↓
  ┌─ 音频预处理（WeNet 语音识别 → 3DMM 表情参数）
  ├─ 人脸检测与对齐（SCRFD）
  ├─ Landmark → 人脸生成
  ├─ 人脸修复（GFPGAN，可选）
  └─ FFmpeg 合成最终视频（支持水印/数字人标识）
        ↓
  输出合成视频文件
```

## 七、AI 功能模块

### 1. AI 文案改写
通过 DeepSeek API 智能改写文案，同时生成视频标题、话题标签、关键词和画中画提示词。
- 桌面端：`unified_app.py` 中的 `_rewrite_text_with_deepseek()` 函数
- 手机端：`hb_ip/utils/api.js` 中的 `aiOptimizeText()` / `aiRewriteText()` 接口

### 2. 法务检查（平台合规检测）
调用 AI 检查文案内容是否符合抖音、小红书、哔哩哔哩、视频号、快手等各大平台的发布规则，自动检测并替换违禁词。
- **检测范围**：极限用词（最、第一、唯一等）、广告法违禁词、诞导性用词、敏感领域违规表述等
- **工作流程**：点击“法务检查”按钮 → AI 检测文案中的违规词 → 弹出对比面板（左侧原文 / 右侧修改结果，红色标注替换的词） → 确认后应用修改
- **核心特点**：只替换违规词语，不改动其他内容；替换词保持原文语义
- 桌面端：`unified_app.py` 中的 `_legal_check_with_deepseek()` 函数，UI 包含 `legal_check_btn` + `legal_check_panel`（对比面板）
- 手机端：`hb_ip/pages/tts/tts.vue` 中的 `doRewriteText()` + 改写弹窗

### 3. AI 标题优化
根据文案内容自动生成吸引人的视频标题和话题标签。
- 桌面端：`unified_app.py` 中的 `_optimize_title_with_deepseek()` 函数

### AI 接口调用
- 统一通过 `_call_deepseek_api()` 调用 DeepSeek API（`https://ai.zhimengai.xyz/v1/chat/completions`）
- 限流：60 秒内只允许调用一次
- 需要配置 `DEEPSEEK_API_KEY`

## 八、模块间通信关系

### 1. TTS 语音合成流程
```
手机端/桌面端 → POST /api/dsp/voice/tts → Hyperf API → LipVoice 云端TTS → 返回音频URL
```

### 2. HeyGem 视频合成流程（在线模式）
```
客户端上传音频+视频 hash → API(HeyGemTaskController)
    ├─ GPU在线 → WebSocket转发 → GPU(run_server.py)执行推理 → 回传结果
    └─ GPU离线 → Redis排队 + 通知开机 → GPU上线后自动批量下发
```

### 3. GPU 自动开关机
```
gpu_power_manager.py ←WebSocket→ Hyperf(Dsp.php)
    ├─ 收到任务通知 → Playwright 操控云控制台开机
    └─ 30分钟无任务 → 自动关机
```

### 4. WebSocket 角色注册
```
Worker(GPU服务器): register { role: "worker" }      → 接收任务，回传结果
Client(PC/手机):   register { key: "xxx" }           → 接收状态推送，提交任务
GPU Monitor:       register { role: "gpu_monitor" }  → 接收任务通知，控制开关机
```
